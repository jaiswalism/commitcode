# Project Instructions & Workflow

This file contains the operating guidelines, workflow rules, and coding standards for the MailVerdict project.

## Workflow Rules

1. **Keep Logs Updated:**
   - After completing a feature or fixing a bug, immediately document it in `CHANGELOG.md`.
   - If a blocker, technical debt, or operational challenge (e.g., seed network issues) is discovered, log it in `ISSUES.md`.
   - Before ending a session or starting a new phase, ensure `CURRENT_FOCUS.md` accurately reflects the immediate next steps.
